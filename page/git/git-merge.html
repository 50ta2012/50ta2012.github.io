合併分支

檢示版本紀錄可以看到 cat 和 master 分別對應著不同的版本，而目前所在的分支(HEAD 所指)為 cat：

$ git log --oneline

5ed08fe (HEAD -> cat) add new file: dog
4500955 add new file: cat
560bb09 (master) add new file: apple
3c68540 first commit

移動到 master 分支：

$ git checkout master

這時候嘗試 merge cat 分支到 master 分支：

$ git merge cat

Updating 560bb09..5ed08fe
Fast-forward
 cat.txt | 1 +
 dog.txt | 1 +
 2 files changed, 2 insertions(+)
 create mode 100644 cat.txt
 create mode 100644 dog.txt

$ git log --oneline

5ed08fe (HEAD -> master, cat) add new file: dog
4500955 add new file: cat
560bb09 add new file: apple
3c68540 first commit

可以看到 master 分支原本在版本號 560bb09 ， merge 之後移動到 5ed08fe 和 cat 分支同一個位置。其實分支就像是書籤，合併分支就如同把書籤 master 放到書籤 cat 所在的版本：
